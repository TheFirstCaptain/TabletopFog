# B-008: Campaign Details Edit Cannot Change Campaign Name

## Summary

The Campaign Library card edit flow is labeled as editing campaign details but
only allows icon and description changes.

## Status

Resolved.

## Observed Behavior

The GM can edit a campaign card icon and description, but cannot edit the
campaign name from the same campaign-details flow.

## Expected Behavior

The GM can edit the campaign display name, icon, and description from the
campaign card edit flow. Renaming should update the name shown on campaign
cards, headings, and breadcrumbs without changing Player Display state.

## Reproduction

1. Open `/gm`.
2. Create or view an existing campaign.
3. Return to the Campaign Library.
4. Click `Edit campaign details`.
5. Confirm only `Campaign icon` and `Campaign description` are editable.

## Affected Systems

- GM view: campaign card edit form.
- Player view: should remain unchanged.
- Local server: GM-only campaign metadata endpoint.
- Map/fog rendering: not affected.
- Storage: campaign display name in `campaign.json`; campaign folder ID should
  remain unchanged.
- Network/device setup: not affected.

## Investigation Notes

F-005A implemented optional campaign card metadata for `description` and `icon`
only. The user-facing edit action is broader than that implementation and makes
the missing name field feel broken.

The fix should update the campaign display name in metadata while preserving
the existing campaign folder ID to avoid a storage-folder migration.

Module baseline growth is expected for this fix because campaign-name metadata
requires one additional submitted GM form field, one rendered GM form field, and
server-side validation for the expanded metadata allowlist.

## Validation Plan

Add storage, server, and Chromium regression coverage for campaign display-name
edits. Confirm invalid name edits are rejected without changing existing
metadata, metadata-only edits remain GM-only, and Player Display asset requests
do not churn.
