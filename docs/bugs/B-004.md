# B-004: Invalid Campaign Metadata Is Silently Hidden

## Summary

Campaign discovery suppresses all read and parse failures, causing affected
campaigns to disappear from the library without an explanation.

## Status

Resolved.

## Observed Behavior

`listCampaigns()` catches campaign errors and returns `null`. The GM receives no
indication that a campaign folder was skipped or how to recover it.

## Expected Behavior

The campaign library should preserve valid campaigns while reporting skipped or
invalid campaign folders with actionable, non-destructive diagnostics.

## Reproduction

1. Create a campaign folder with malformed or unreadable `campaign.json`.
2. Load the campaign library.
3. Observe that the campaign is omitted without a warning.

## Affected Systems

- GM view: campaign discovery and diagnostics.
- Local server: error reporting.
- Storage: recovery from invalid metadata.

## Investigation Notes

The current behavior keeps one invalid folder from breaking the whole library,
which should be preserved. The missing behavior is visibility and recovery
guidance.

Reproduced on 2026-06-20 by placing malformed JSON in one campaign folder next
to a valid campaign. `listCampaigns()` returned the valid campaign and silently
dropped the malformed folder; the API response contained no warning field and
the GM library had no diagnostic surface. The invalid file remained unchanged.

The fix will retain the existing campaign array for compatibility and add a
safe diagnostic containing only the skipped folder ID plus generic recovery
guidance. Raw parser errors and filesystem paths will not be exposed.

The minimal fix adds bounded responsibility-aligned code to three modules that
already have named extraction plans: campaign discovery in
`server/campaign-storage.js` under E-002, route response composition in
`server/index.js` under E-004, and GM library rendering in `public/gm.js` under
E-003. Their exact module baselines are intentionally raised for this bug fix;
the existing engineering items remain responsible for later reductions.

## Validation Plan

Add tests for mixed valid and invalid campaigns, expose a safe diagnostic to the
GM, and verify invalid files are not modified automatically.

## Resolution

Campaign discovery now returns valid campaign summaries plus a separate sorted
diagnostic list for unreadable or invalid folders. The API preserves the
existing `campaigns` array and adds `diagnostics`; the GM library renders each
diagnostic as a visible recovery message while keeping valid campaigns usable.

Diagnostics contain only the skipped folder ID and generic guidance to fix or
restore `campaign.json`. They do not expose parser errors or absolute paths, and
render through `textContent`. Discovery remains read-only and does not repair,
rewrite, or delete invalid metadata.
