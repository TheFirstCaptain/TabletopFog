# B-004: Invalid Campaign Metadata Is Silently Hidden

## Summary

Campaign discovery suppresses all read and parse failures, causing affected
campaigns to disappear from the library without an explanation.

## Status

Open.

## Observed Behavior

`listCampaigns()` catches campaign errors and returns `null`. The GM receives no
indication that a campaign folder was skipped or how to recover it.

## Expected Behavior

The campaign library should preserve valid campaigns while reporting skipped or
invalid campaign folders with actionable, non-destructive diagnostics.

## Reproduction

1. Create a campaign folder with malformed or unreadable `campaign.json`.
2. Load the campaign library.
3. Observe that the campaign is omitted without a warning.

## Affected Systems

- GM view: campaign discovery and diagnostics.
- Local server: error reporting.
- Storage: recovery from invalid metadata.

## Investigation Notes

The current behavior keeps one invalid folder from breaking the whole library,
which should be preserved. The missing behavior is visibility and recovery
guidance.

## Validation Plan

Add tests for mixed valid and invalid campaigns, expose a safe diagnostic to the
GM, and verify invalid files are not modified automatically.
