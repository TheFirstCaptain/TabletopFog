# B-009: Encounter Workspace Narrows After Opening Encounter

## Summary

Opening an encounter could leave the focused workspace constrained to the old
right-side gallery column instead of using the available campaign width.

## Status

Resolved.

## Observed Behavior

The initial campaign layout looked correct, but after clicking an encounter card
the workspace became a narrow column with substantial unused space to the right.

## Expected Behavior

Opening an encounter should switch to the focused workspace layout, with the
workspace spanning the campaign content area and the map remaining prominent.

## Reproduction

1. Open a campaign with at least one encounter.
2. Observe the initial campaign gallery and empty workspace prompt.
3. Click an encounter card to open it for prep.
4. Observe that the workspace content narrows instead of filling the campaign
   content area.

## Affected Systems

- GM view: focused encounter workspace layout.
- Player view: no change.
- Local server: no change.
- Map/fog rendering: no renderer behavior change.
- Storage: no change.
- Network/device setup: no change.
- Documentation: bug tracker updated.

## Investigation Notes

The gallery and workspace share `.workspace-grid`. When the workspace opened,
the gallery was hidden but the parent grid still reserved the gallery column, so
`#encounter-workspace` remained constrained to the second column. The fix adds an
explicit `data-workspace-open` layout state and lets the open workspace span the
available grid width.

## Validation Plan

Add Chromium regression assertions that the opened workspace spans its parent
grid at desktop and Chromebook-sized widths. Run focused browser validation and
the full quality gate.
