# B-012: Brush Preview And Size Layout Are Hard To Use

## Summary

Brush mode does not show the circular brush outline until the GM presses on the
map, and the Brush diameter number sits to the right of the slider where it
looks cramped.

## Status

Resolved.

## Observed Behavior

After choosing Brush and turning on Hide or Reveal, hovering over the map does
not show the brush footprint. The Brush diameter range input, numeric field,
and percent sign also render on one row, which makes the value feel visually
detached from the control.

## Expected Behavior

When Brush and a fog action are active, moving the pointer over the map should
show the circular brush outline before painting starts. The Brush diameter
numeric value should sit below the slider with the percent sign.

## Reproduction

1. Open a campaign with an encounter map in the GM View.
2. Open the encounter workspace.
3. Choose `Brush`.
4. Choose `Hide` or `Reveal`.
5. Move the pointer over the map without pressing.
6. Inspect the Brush diameter control layout.

## Affected Systems

- GM view: brush preview and Brush diameter control layout.
- Player view: not affected.
- Local server: not affected.
- Map/fog rendering: draft preview only; committed fog operations are unchanged.
- Storage: not affected.
- Network/device setup: not affected.
- Documentation: bug tracker only.

## Investigation Notes

Brush preview was only updated during an active pointer drag. The size control
used a single-row grid for the range input, number input, and percent sign.

Fix summary:

- Brush pointer hover now previews the circular brush footprint before painting.
- Leaving the brush overlay clears an uncommitted hover preview.
- The Brush diameter slider stays aligned with the Fog controls, with the
  number and percent sign below it.
