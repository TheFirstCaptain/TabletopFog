# B-007: Data Root Path Overflows Narrow GM Layout

## Summary

The GM campaign library can overflow horizontally at narrow viewport widths
when the displayed local campaign data path is longer than the available panel.

## Status

Resolved.

## Observed Behavior

At a 390 px Chromium viewport, the temporary campaign data path renders as one
unbroken segment and makes the document wider than the viewport.

## Expected Behavior

The data-root path wraps within the campaign panel without changing the path
text or causing horizontal page overflow.

## Reproduction

1. Start the app with a long `TABLETOPFOG_DATA_DIR` path.
2. Open `/gm` at a 390 px viewport.
3. Observe that the data-root text extends past the panel and the document
   scroll width exceeds the viewport width.

F-004A narrow-layout coverage reproduced this on 2026-06-21 before the theme
implementation's responsive validation could pass.

## Affected Systems

- GM view: campaign-library data-root display at narrow widths.
- Storage: path display only; stored data and configured path are unchanged.
- Tests and tooling: Chromium narrow-layout regression.

## Investigation Notes

The path is intentionally displayed as plain text. The narrow fix should allow
breaking inside the existing data-root element without truncating, hiding, or
changing its content.

## Validation Plan

Retain the failing 390 px Chromium overflow assertion, allow wrapping on the
data-root element, rerun the focused theme suite and existing workflow suite,
and run `npm run quality`.

## Resolution

The data-root element now permits wrapping at arbitrary points when its path
cannot otherwise fit the available width. The displayed path is unchanged.

## Validation Evidence

- The initial 390 px F-004A theme test failed because document scroll width
  exceeded viewport width.
- The focused post-fix Chromium test passed with the same real temporary path.
- `npm run quality` passed all 6 Chromium workflow and theme tests, all 75 Node
  tests with coverage floors, lint, formatting, module and harness policy, and
  the high-severity dependency audit.
