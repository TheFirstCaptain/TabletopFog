# B-001: WebSocket Dependency Permits Memory-Exhaustion DoS

## Summary

The installed Socket.IO dependency chain includes a vulnerable `ws` release
affected by a memory-exhaustion denial-of-service advisory.

## Status

Open.

## Observed Behavior

On 2026-06-20, `npm audit` reported four high-severity affected package entries
through `ws` version `8.20.1`. A fix is available.

## Expected Behavior

Installed runtime and development dependencies should have no known high or
critical vulnerabilities with available compatible fixes.

## Reproduction

1. Install dependencies from `package-lock.json`.
2. Run `npm audit`.
3. Observe the high-severity `ws` advisory and affected Engine.IO/Socket.IO
   packages.

## Affected Systems

- Local server: Socket.IO WebSocket transport.
- Tests and tooling: Socket.IO client transport.
- Network/device setup: exposure is limited by the trusted-LAN deployment model
  but is not eliminated.

## Investigation Notes

The advisory is `GHSA-96hv-2xvq-fx4p`. The repository binds the server to the LAN
by default, so this remains relevant even though internet hosting is out of
scope.

## Validation Plan

Update the compatible dependency chain, run `npm test`, run `npm audit`, and
perform a local Socket.IO connection smoke test.
