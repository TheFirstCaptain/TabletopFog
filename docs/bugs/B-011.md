# B-011: Circle Fog Tool Behaves Like A Brush

## Summary

The GM fog tool labeled `Circle` behaves like a fixed-size placement tool
instead of a drag-to-size circle shape. This is misleading because `Rectangle`
is drag-to-size, while the fixed-size circular control is better understood as
a brush.

## Status

Resolved.

## Observed Behavior

The GM chooses `Circle`, sets a diameter, and taps the map to place one fixed
circle. Dragging cancels placement instead of resizing the circle or painting
fog.

## Expected Behavior

The fixed-size circular tool should be labeled `Brush`, keep its size control,
and support freehand drag painting. A separate drag-to-size `Circle` shape is
tracked as F-008E.

## Reproduction

1. Open a campaign with an encounter map in the GM View.
2. Open the encounter workspace.
3. Choose `Hide`, then choose `Circle`.
4. Drag across the map.

## Affected Systems

- GM view: misleading fog tool label and drag behavior.
- Player view: receives the resulting fog operations only after successful GM
  commits.
- Local server: no server behavior change expected; brush uses existing circle
  fog operations.
- Map/fog rendering: brush stamps render through existing circle replay.
- Storage: no storage shape change expected; brush stamps persist as existing
  `hide-circle` and `reveal-circle` operations.
- Network/device setup: not affected.
- Documentation: README, acceptance tests, tracker, and related feature notes
  should describe Brush separately from the future drag-to-size Circle shape.

## Investigation Notes

F-008D implemented fixed-size circular fog with click/tap placement and
explicitly deferred brush and drag-to-size behavior. The user later clarified
that the fixed-size circular behavior feels like a brush and that a true Circle
shape should work like Rectangle.

The existing server schema and renderer already support ordered
`hide-circle` and `reveal-circle` operations, so the correction can reuse the
current persistence and rendering path without migration.

Fix summary:

- Renamed the visible fixed-size circular fog tool to `Brush`.
- Brush size controls now say `Brush diameter`.
- Brush click/tap still creates one circular fog operation.
- Brush drag paints multiple circular fog operations along the drag path.
- Rectangle behavior is unchanged.
- A true drag-to-size Circle shape is tracked separately as F-008E.
