# B-003: Map Upload Accepts Invalid Image Data

## Summary

The map upload endpoint persists arbitrary non-empty bytes without verifying
that they represent a supported image.

## Status

Open.

## Observed Behavior

The browser supplies a content type and file name, but the server does not use
the collected content type or inspect the file. Invalid data can be stored as a
map and later fail in the GM or player image element.

## Expected Behavior

Map uploads should reject unsupported or invalid image data with a clear error
before campaign metadata is updated.

## Reproduction

1. Send arbitrary non-empty bytes to the map upload API.
2. Supply an image content type and image filename.
3. Observe that the map is persisted successfully.

## Affected Systems

- GM view: map upload workflow and error reporting.
- Player view: active-map image rendering.
- Local server: upload validation and memory use.
- Storage: invalid map assets and metadata.

## Investigation Notes

The endpoint currently buffers up to 100 MB and test fixtures use string data
labeled as PNG. Supported formats and validation depth require clarification
before implementation.

## Validation Plan

Define supported image formats and size behavior, add valid and invalid upload
tests, verify rejected uploads leave no files or metadata, and manually validate
representative browser uploads.
