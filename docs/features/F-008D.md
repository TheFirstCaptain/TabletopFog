# F-008D: Expanded Fog Shape Tools

## Objective

Explore and define fog tools beyond rectangles, starting with circular hide and
reveal areas whose size can be selected quickly or entered precisely.

## User Value

The GM can hide or reveal round rooms, spell areas, cave sections, and irregular
table moments with less manual rectangle stacking while keeping fog editing
fast during play.

## Non-Goals

- Do not add tokens, vision cones, dynamic lighting, line of sight, or
  automation.
- Do not add a full drawing program or arbitrary shape editing.
- Do not add player controls.
- Do not change which encounter is `Shown to Players`.
- Do not replace the existing rectangle Hide and Reveal tools.
- Do not add cloud storage, accounts, or shared online play.
- Do not migrate storage/API terminology such as `maps`, `activeMapId`, or
  `fog`.

## Acceptance Notes

- F-008D V1 adds circular fog tools only; brush-like tools remain out of scope.
- Hide and Reveal remain the primary fog actions.
- Rectangle and Circle are shape choices inside the Fog tools group.
- Rectangle drawing keeps the existing drag-to-draw behavior.
- Circle drawing is click/tap-to-place using the currently selected diameter;
  pointer movement beyond the tap threshold cancels placement rather than
  committing a circle.
- Circle diameter is expressed as a percent of the map's shorter side so saved
  circle operations are stable across GM zoom and viewport changes.
- Circle size can be adjusted with a slider and typed directly into an input.
- Circle size controls must support fine-grained values rather than limiting
  the GM to coarse 25% increments.
- Circle diameter V1 range is 2 through 100 percent, default 20 percent, step 1
  percent.
- Preset sizes are not included in V1.
- Circle supports both hide and reveal behavior.
- Circle operations persist as ordered operations:
  `{ type: "hide-circle", circle: { x, y, radius } }` and
  `{ type: "reveal-circle", circle: { x, y, radius } }`, where `x` and `y` are
  normalized map center coordinates and `radius` is normalized against the
  shorter map dimension.
- Circles may clip at map edges; the stored center must remain on the map.
- New fog tools live in the Encounter Workspace Fog tools group.
- Shape tools must respect the selected/editing versus `Shown to Players`
  distinction.
- If the selected encounter is currently `Shown to Players`, completed fog
  changes update the Player Display live.
- If the selected encounter is not `Shown to Players`, completed fog changes do
  not change the Player Display.
- New fog operations autosave to local campaign storage.
- Existing rectangle operations, undo behavior, and Clear Fog behavior must
  remain compatible.
- Player Display remains read-only.
- The UI must stay calm and usable for a standing GM; avoid dense toolbars or
  precision-only controls.

## SDLC Tier

Tier: 3.

Rationale: Expanded fog tools are expected to affect GM fog input, persisted fog
operation shapes, filesystem writes, server state, Socket.IO synchronization,
Player Display projection, player read-only guarantees, and map/fog rendering
semantics. Those areas are Tier 3 under
`docs/decisions/decision-004-mandatory-subagent-sdlc.md`.

## Affected Systems

- GM view: adds Hide/Reveal action buttons, Rectangle/Circle shape buttons, and
  Circle diameter slider plus typed input in the Encounter Workspace Fog tools
  group.
- Player view: receives rendered circle fog changes only for the encounter
  currently `Shown to Players`.
- Local server: accepts and validates `hide-circle` and `reveal-circle`
  operations through the existing GM-only fog operation route.
- Map/fog rendering: replays ordered rectangle and circle operations into the
  same fog mask for GM and Player Display.
- Storage: preserves and recovers persisted circle operations in local campaign
  JSON.
- Documentation: roadmap, tracker, acceptance tests, and README current status
  reflect circle fog tools.
