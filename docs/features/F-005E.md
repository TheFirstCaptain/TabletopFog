# F-005E: Campaign Card Presentation

## Objective

Improve the visual presentation of campaign cards.

## User Value

The Campaign Library feels calmer, easier to scan, and more like browsing a
small tabletop collection than managing records.

## Non-Goals

- Do not change encounter cards.
- Do not change the Encounter Workspace.
- Do not change storage.
- Do not add new functionality.
- Do not change campaign creation or metadata behavior.

## Acceptance Notes

- Campaign card typography is consistent with the design language.
- Card spacing supports quick scanning without crowding.
- Campaign icon placement is visually balanced.
- Description layout is readable and does not dominate the card.
- Metadata layout is quiet and secondary to campaign identity.
- Hover and focus behavior are clear, accessible, and restrained.
- Responsive spacing remains usable on Chromebook-sized and narrow viewports.
- Campaign cards remain cards, not tables or dashboard widgets.

## Feature Readiness Checklist

## Clarification Notes

- This is a focused visual presentation feature for Campaign Library cards only.
- Do not bundle navigation cleanup, encounter-card work, or workspace layout
  changes into this feature.
- Align with `docs/design-language.md`: Calm Interface, Cards Over Tables,
  Designed For A Standing GM, and Rule of Three.
- Campaign descriptions should use a fixed two-line clamp so a long description
  remains readable without dominating the card grid.
- Campaign icons remain emoji or short-symbol only for this feature. Uploaded
  campaign images and search/filtering remain deferred.
- This feature is visual polish only. It should change card layout, hierarchy,
  and presentation without changing campaign creation, metadata persistence, or
  Player Display behavior.
- Responsive validation should cover Chromebook-sized and narrow viewports.
