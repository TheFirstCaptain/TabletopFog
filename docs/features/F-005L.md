# F-005L: Delete Encounters

## Objective

Let the GM permanently delete an encounter from a campaign after confirmation.

## User Value

The GM can remove test or unwanted encounters from the campaign without waiting
for a future archive workflow, while protected running/prep states prevent
accidental table disruption.

## Non-Goals

- Do not delete campaigns in this feature.
- Do not add archive, trash, undo, restore, or soft-delete behavior.
- Do not add campaign reordering or campaign cleanup behavior.
- Do not allow deleting the encounter currently `Shown to Players`.
- Do not add Player Display controls.
- Do not add fog deletion or future encounter-tool deletion beyond removing the
  encounter record and its current map asset.

## Acceptance Notes

- Delete controls are available only in Manage Mode.
- Clicking delete requires a confirmation before any data is removed.
- Canceling the confirmation leaves the encounter, map asset, order, selected
  encounter, and Player Display unchanged.
- Confirming delete permanently removes an encounter and its map asset from
  local storage when that encounter is not currently `Shown to Players`.
- The remaining encounter order is repaired after deletion.
- The GM cannot delete the encounter currently `Shown to Players`.
- The GM can delete an encounter currently `Selected for Prep` as long as it is
  not currently `Shown to Players`.
- Rejected delete attempts preserve campaign metadata, map files, selected prep
  state, and Player Display state.

## Clarification Notes

- The user requested delete behavior before moving to polish features.
- Deletes require confirmation.
- Deletes are permanent for now because current campaign data is test data.
- Archive, restore, and campaign reordering can be considered later to reduce
  clutter.
- An encounter cannot be deleted while it is shown to players.
- Follow-up clarification: selected-for-prep status should not block deletion.
  This allows deleting the only encounter in a campaign when it is automatically
  selected for prep, as long as it is not shown to players.
- Campaign deletion should be handled separately because it removes a whole
  campaign folder and has different safety rules from encounter deletion.
