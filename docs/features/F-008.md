# F-008: Table Quality-of-Life Improvements

## Objective

Improve the table experience after campaign library, active map display, fog, and save/load are proven.

## User Value

The GM can run sessions with less friction once the core V1 workflow is stable.

## Non-Goals

- Do not convert TabletopFog into a full VTT.
- Do not add initiative tracking.
- Do not add encounter tracker integration.
- Do not add notes.
- Do not add character sheets.
- Do not add dice rolling.
- Do not add tokens or NPC tracking.
- Do not add dynamic lighting or secret room layers.
- Do not add cloud dependencies.

## Acceptance Notes

- Complete: expanded fog shape tools in F-008D.
- Complete: undo fog action.
- Proposed: clear or reset fog for the current map beyond the existing
  per-encounter Clear Fog action if table use shows a gap.
- Complete: fullscreen player view.
- Complete: copied Player Display URL.
- Deferred: QR code generation.
- Better GM map controls if needed.
- Split this into child features before implementation if it grows too large.

## Child Features

- [F-008A](./F-008A.md): Player URL sharing.
- [F-008B](./F-008B.md): Fullscreen Player View.
- [F-008C](./F-008C.md): Undo Fog Action.
- [F-008D](./F-008D.md): Expanded Fog Shape Tools.

## Affected Systems

- GM view: improved table controls.
- Player view: fullscreen and display polish.
- Local server: may expose player URL details.
- Map/fog rendering: circle fog shape tools, with brush and reset behavior
  deferred until table use shows a gap.
- Storage: completed undo autosaves the resulting fog state while keeping undo
  history runtime-only.
- Documentation: split into child features before implementation if scope grows.

## Validation Plan

Validate each improvement independently with manual GM/player checks and update automated tests when behavior is stable.

## Open Questions

- Resolved: Player URL sharing and Fullscreen Player View were the first
  quality-of-life child features after save/load.
- Deferred: QR code generation should be designed in a later child feature if
  copied URLs are not enough for table setup.
- Resolved: F-008C uses repeatable single-step fog undo with runtime-only
  history.
- Resolved: F-008D starts with circle fog shape tools using typed and slider
  diameter controls; brush behavior and additional fog shapes remain deferred
  until table use shows a gap.

## Status
