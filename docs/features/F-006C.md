# F-006C: Rectangle Hide Tool

## Objective

Add the first GM-facing fog editing tool: drawing rectangular hidden areas on
the selected encounter map.

## User Value

The GM can quickly hide unrevealed rooms, corridors, or map sections during prep
or play without using a full VTT feature set.

## Non-Goals

- Do not add reveal drawing; that is F-006D.
- Do not add clear fog; that is F-006E.
- Do not add circles, brushes, polygon tools, undo, or shape editing.
- Do not persist fog to campaign storage.
- Do not add player controls.
- Do not add dynamic lighting, tokens, vision cones, notes, or initiative.

## Acceptance Notes

- Rectangle Hide controls live in the Encounter Workspace.
- The GM can choose Hide mode and draw a rectangle over the selected encounter
  map.
- Completing the drag appends a `hide-rectangle` operation to that encounter's
  in-memory fog list.
- The GM sees the hidden rectangle semi-transparently.
- If the selected encounter is shown to players, the Player Display updates live
  with opaque or near-opaque fog.
- If the selected encounter is not shown to players, the Player Display does not
  change.
- Hide drawing respects the current GM map zoom and map positioning.
- Opening or editing an encounter still does not show it to players.
- `Show to Players` remains the only action that changes which encounter the
  Player Display shows.

## Clarification Notes

- 2026-07-01: User confirmed rectangle support is acceptable for the first fog
  tool slice.
- 2026-07-01: User agreed fog remains in-memory until F-007.

## Affected Systems

- GM view: add Rectangle Hide tool controls and pointer interaction.
- Player view: receive live fog updates only for the shown encounter.
- Local server: accept and sync hide operations for relevant encounter state.
- Map/fog rendering: convert GM drag geometry into map-relative fog operations.
- Storage: no persistent storage changes.
- Documentation: update F-006 child-feature evidence and acceptance tests.

## Validation Plan

Draw rectangular hide areas on an unshown selected encounter and confirm players
do not change. Show that encounter to players, draw another hide rectangle, and
confirm the Player Display updates live. Repeat with GM zoom/grid state active
to confirm drawing maps to the correct map area.

## Open Questions

- Should Escape cancel an in-progress rectangle drag in the first version?
