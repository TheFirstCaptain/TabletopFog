# F-005O: One-Line GM App Header

## Objective

Compress the GM View app-level header into one line so it uses less vertical
space before the campaign, encounter gallery, or Encounter Workspace content.

## User Value

The GM sees more of the working page immediately, with app identity,
navigation context, and connection state still visible.

## Non-Goals

- Do not change Campaign Library, encounter gallery, or Encounter Workspace
  workflows.
- Do not change Player Display behavior.
- Do not change storage, API routes, campaign metadata, or Socket.IO state.
- Do not add fog controls or new navigation actions.
- Do not change the Player Display status bar.
- Do not add animations or transitions.

## Acceptance Notes

- GM View app header renders as one compact horizontal line on desktop and
  Chromebook-sized screens.
- `TABLETOPFOG` remains on the left, uses the old all-caps system-font brand
  treatment, and remains the
  page's primary heading.
- The separate dynamic page-title line, such as `Campaign`,
  `Campaign Library`, or `Encounter Workspace`, is removed from the app header.
- Breadcrumb text appears after `TABLETOPFOG`.
- Connection state such as `Live` or `Reconnecting...` appears on the same row,
  aligned to the right.
- Long breadcrumb text does not cause horizontal overflow.
- Existing Back actions remain the obvious way to navigate.
- No Player Display, storage, API, map rendering, or workflow behavior changes.
