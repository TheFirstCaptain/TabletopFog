# F-005F: Encounter Gallery Presentation

## Objective

Make the encounter gallery feel like browsing encounters instead of managing
files.

## User Value

The GM can visually recognize and open prepared encounters quickly, with the map
thumbnail and encounter identity carrying the screen instead of administrative
controls.

## Non-Goals

- Do not change the Encounter Workspace.
- Do not add fog.
- Do not add Manage Mode.
- Do not add new functionality.
- Do not change storage, upload behavior, rename behavior, or reorder behavior.

## Acceptance Notes

- Encounter thumbnails are larger and more prominent.
- Card spacing improves scanning and avoids clutter.
- Visual hierarchy emphasizes encounter name, thumbnail, and shown/editing
  status.
- Status indicators are cleaner and use `Shown to Players` consistently.
- Add Encounter presentation is clearer without dominating the gallery.
- Administrative controls do not visually compete with browsing.
- Card layout remains responsive and touch-friendly.
- Opening an encounter still does not change the Player Display.

## Clarification Notes

- This feature improves the existing encounter gallery presentation without
  introducing Manage Mode. F-005H may later move administrative controls behind
  a dedicated mode.
- Align with `docs/design-language.md`: Maps First, Calm Interface, Cards Over
  Tables, and Never Surprise the GM.
- Keep `Show to Players` as the explicit player-display action.

## Affected Systems

- GM View: Campaign encounter gallery cards, Add Encounter presentation, and
  status indicators.
- Player Display: no intended changes except existing explicit `Show to Players`
  behavior.
- Local server: no intended changes.
- Map/fog rendering: no intended changes.
- Storage: no intended changes.
- Documentation: update acceptance notes if gallery presentation expectations
  change.

## Validation Plan

Validate a campaign with zero, one, and multiple encounters, including long
encounter names and varied thumbnail aspect ratios. Confirm browsing, opening,
rename, reorder, upload, and explicit `Show to Players` behavior still work,
and opening an encounter still leaves the Player Display unchanged.
