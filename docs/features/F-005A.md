# F-005A: Campaign Landing Page Polish

## Objective

Replace the utilitarian campaign list with a more polished campaign landing page using cards.

## User Value

The GM starts from a lightweight fantasy campaign shelf that is easier to scan before a session while keeping the app focused on local prep and table display.

## Non-Goals

- Do not add player or member management.
- Do not add campaign notes.
- Do not add character sheets.
- Do not add cloud sync.
- Do not add decorative image assets unless explicitly approved.
- Do not redesign storage beyond metadata needed for campaign description or icon.
- Do not add fog controls.
- Do not add player tokens.
- Do not add VTT-style navigation.

## Acceptance Notes

- GM `/gm` starts on a campaign landing page.
- Campaigns display as cards, not a plain list.
- Each campaign card shows the campaign name.
- Each campaign card shows an editable short description when available.
- Each campaign card shows an emoji icon.
- Campaign cards may show basic metadata such as map or encounter count, or last modified, only when available without heavy new work.
- Clicking a campaign card opens that campaign.
- New campaign creation remains available.
- Campaign search or filtering is deferred to future polish.
- The page keeps the F-004A fantasy/parchment visual theme.
- The layout works on Chromebook-sized screens.
- Invalid campaign diagnostics remain visible and usable.

## Terminology Notes

- Use "campaign" for the top-level local folder and prep bucket.
- Prefer lightweight fantasy-tabletop language over full-VTT concepts such as parties, members, characters, or campaign dashboards.
- Store campaign emoji icons and descriptions as optional campaign metadata
  while preserving compatibility with existing campaign folders.

## Clarification Notes

- Use emoji icons for F-005A campaign cards.
- Defer uploaded campaign images to a future feature.
- Make campaign descriptions editable in F-005A.
- Defer campaign search or filtering to future polish.
- Keep this scoped to the GM campaign landing page; do not change player view behavior.
- Preserve existing campaign storage compatibility by treating description and icon metadata as optional fields.
- Use a short free-form emoji/symbol text field for the icon in F-005A, with a
  default map icon when the field is blank or missing.
- Limit descriptions to 160 characters; blank descriptions are allowed.
