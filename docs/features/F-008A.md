# F-008A: Player URL Sharing

## Objective

Give the GM a quiet way to see and copy the current Player Display URL from the
GM View.

## User Value

The GM can get the player-facing URL quickly during table setup without reading
terminal output or manually changing `/gm` to `/player`.

## Non-Goals

- Do not add QR code generation.
- Do not add LAN IP detection or certificate regeneration.
- Do not add a server API for URL discovery.
- Do not add accounts, access tokens, or authentication.
- Do not expose GM controls on the Player Display.
- Do not change what is shown to players.

## Acceptance Notes

- The Campaign Library shows a compact GM-only Player Display URL area.
- The displayed URL is derived from the GM page's current origin and `/player`.
- The GM can copy the displayed Player Display URL.
- Copy success or failure is reported quietly in GM-only status text.
- Opening, copying, or viewing the URL does not change the Player Display.
- No QR code is included in this slice.
