# F-006D: Rectangle Reveal Tool

## Objective

Add rectangular reveal drawing so the GM can remove visible fog from hidden
areas during play.

## User Value

The GM can reveal explored rooms, halls, or sightlines at the table while
preserving other hidden areas.

## Non-Goals

- Do not add circles, brushes, polygon tools, undo, or shape editing.
- Do not add clear fog; that is F-006E.
- Do not persist fog to campaign storage.
- Do not normalize or rewrite the full fog shape list.
- Do not add dynamic lighting, tokens, vision cones, notes, or initiative.

## Acceptance Notes

- Rectangle Reveal controls live in the Encounter Workspace beside the Hide
  tool.
- The GM can choose Reveal mode and draw a rectangle over the selected encounter
  map.
- Completing the drag appends a `reveal-rectangle` operation to that encounter's
  in-memory fog list.
- Rendering replays operations in order, so reveal rectangles cut through
  previously hidden areas.
- A later hide rectangle can cover an area revealed earlier.
- If the selected encounter is shown to players, the Player Display updates live
  with that area revealed.
- If the selected encounter is not shown to players, the Player Display does not
  change.
- Reveal drawing respects the current GM map zoom and map positioning.
- Hide and Reveal are separate explicit buttons, and only one fog drawing action
  can be active at a time.
- Reveal drawing reuses Hide's tiny-drag filtering, Escape cancellation,
  off-map-start rejection, and grid input arbitration.
- Player Display remains read-only.
