# F-009C: Continuous Integration Enforcement

## Objective

Run the authoritative local quality checks automatically on proposed repository
changes using a supported Node environment.

## User Value

Quality expectations are enforced consistently even when an agent or
contributor forgets to run a local check.

## Non-Goals

- Do not create different local and CI quality policies.
- Do not deploy or host the application.
- Do not add cloud runtime dependencies to TabletopFog.
- Do not silently ignore unavailable or flaky required checks.

## Acceptance Notes

- CI installs dependencies from the lockfile using the documented Node version.
- CI runs the same aggregate quality command used locally.
- Required failures block successful completion.
- Network-dependent checks have an explicit failure and exception policy.
- CI configuration is minimal, documented, and contains no application secrets.
- The local command remains sufficient for contributors without CI access.

## Clarification Notes

- Use GitHub Actions because the repository is hosted on GitHub.
- Validate the documented minimum with Node.js 22.8.0 and the current runtime
  line with Node.js 24.
- Run on every pull request and every push to `main`.
- Run the complete `npm run quality` command for every matrix entry without
  changed-file scoping.
- Keep the local and CI quality policies identical; CI may only add setup and
  trigger configuration around the authoritative local command.
