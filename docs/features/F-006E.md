# F-006E: Clear Fog

## Objective

Add an explicit GM control to clear all fog operations from the selected
encounter.

## User Value

The GM can recover quickly from prep mistakes or reset a map without deleting
and recreating the encounter.

## Non-Goals

- Do not add undo history or per-operation deletion.
- Do not add reset-to-saved-state behavior.
- Do not persist fog to campaign storage.
- Do not add player controls.
- Do not add dynamic lighting, tokens, vision cones, notes, or initiative.

## Acceptance Notes

- Clear Fog is available only in the Encounter Workspace for the selected
  encounter.
- Clear Fog removes all in-memory fog operations for that encounter.
- Clear Fog is explicit and confirmed before removing fog.
- If the selected encounter is shown to players, clearing fog updates the Player
  Display live.
- If the selected encounter is not shown to players, clearing fog does not
  change the Player Display.
- Clearing fog does not change which encounter is shown to players.
- Player Display remains read-only.

## Clarification Notes

- 2026-07-01: User confirmed they want a Clear Fog control as part of the F-006
  child-feature sequence.

## Affected Systems

- GM view: add confirmed Clear Fog action.
- Player view: receive live cleared-fog state only for the shown encounter.
- Local server: clear selected encounter's in-memory fog operation list.
- Map/fog rendering: render no fog when the list is empty.
- Storage: no persistent storage changes.
- Documentation: update acceptance tests and F-006 completion notes.

## Validation Plan

Create fog on the selected encounter, use Clear Fog, confirm the GM overlay is
removed, and confirm the Player Display updates only when that encounter is
shown to players. Confirm cancellation leaves fog unchanged.

## Open Questions

- Should Clear Fog be visible at all times in the fog tool group or only after
  the selected encounter has at least one fog operation?
