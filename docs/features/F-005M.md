# F-005M: Delete Campaigns

## Objective

Let the GM permanently delete an empty campaign from the Campaign Library after
confirmation.

## User Value

The GM can remove test or unwanted campaign folders once their encounters have
already been cleaned up, without risking accidental deletion of prepared maps.

## Non-Goals

- Do not delete campaigns that still contain encounters or map assets.
- Do not add archive, trash, undo, restore, or soft-delete behavior.
- Do not add bulk campaign cleanup or campaign reordering.
- Do not add Player Display controls.
- Do not rename local campaign folders.

## Acceptance Notes

- Campaign cards expose a secondary `Delete...` action from the Campaign
  Library.
- Campaign delete requires confirmation before any local files are removed.
- Canceling the confirmation leaves the campaign folder, metadata, library card,
  and Player Display unchanged.
- Confirming delete permanently removes an empty campaign folder from local
  storage.
- Campaigns with one or more encounters cannot be deleted.
- Blocked campaign delete controls show a visible reason.
- The server enforces the empty-campaign rule and GM-only access even if a
  stale or handcrafted request is sent.
