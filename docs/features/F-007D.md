# F-007D: Persistence Status and Diagnostics

## Objective

Add minimal GM-facing save/load status or diagnostics only if the earlier F-007
slices show that visible persistence feedback is needed.

## User Value

The GM can tell whether local persistence is working when running a session or
recovering from storage errors.

## Non-Goals

- Do not add manual save workflows unless explicitly approved.
- Do not add cloud sync, accounts, or collaboration.
- Do not expose internal file paths in normal UI.
- Do not add broad settings or admin screens.

## Acceptance Notes

- Any status text is quiet, compact, and GM-only.
- Normal successful autosave should not distract from the map.
- Diagnostics should be actionable when persistence fails.
- Player Display remains read-only and does not expose storage details.
