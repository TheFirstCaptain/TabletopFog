# F-001: Documentation Harness and Repo Structure

## Objective

Create the initial documentation and validation harness for TabletopFog before application code begins.

## User Value

Future development sessions have a shared source of truth for the project vision, scope boundaries, architecture, roadmap, acceptance tests, and AI workflow.

## Non-Goals

- Do not build the application.
- Do not choose final implementation details beyond the documented recommended direction.
- Do not add cloud hosting, accounts, tokens, dice, sheets, or rules automation.

## Acceptance Notes

- `docs/vision.md` defines the product purpose and MVP boundaries.
- `docs/architecture.md` describes the browser-first local-hosting model.
- `docs/roadmap.md` lists Milestones 0-5 and future ideas out of MVP.
- `docs/acceptance-tests.md` defines human-readable acceptance tests.
- `docs/ai-harness.md` defines future AI session rules.
- `docs/decisions/` contains the initial decision records.
- `AGENTS.md` exists as a contributor guide.

## Affected Systems

- GM view: not started.
- Player view: not started.
- Local server: not started.
- Map/fog rendering: not started.
- Storage: not started.
- Documentation: initial harness created.

## Validation Plan

Validate by documentation review and file-tree inspection.
