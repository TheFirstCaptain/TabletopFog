# F-009D: Browser Workflow Coverage

## Objective

Add automated browser characterization for the current critical GM and player
workflows before client refactoring and display behavior expand.

## User Value

Maintainers can refactor and extend browser code with evidence that campaign and
player workflows still behave correctly in a real DOM and browser environment.

## Non-Goals

- Do not redesign the GM or player UI.
- Do not replace physical iPad, TV, or same-Wi-Fi validation.
- Do not add fog behavior or F-005 display polish.
- Do not introduce a frontend framework or application build system.

## Acceptance Notes

- Browser tests cover campaign creation and open behavior.
- Browser tests cover valid upload and rejected invalid upload behavior.
- Browser tests cover rename, reorder, and active-map selection.
- Browser tests confirm the player follows active-map synchronization.
- Browser tests cover connection state and image-load error behavior where
  deterministic automation is practical.
- Browser tests confirm the player exposes no mutation controls.
- Browser tests run through the F-009A local quality command and F-009C CI.
- Physical device checks remain documented separately.
