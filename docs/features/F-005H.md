# F-005H: Manage Mode

## Objective

Separate browsing from administration.

## User Value

The GM can browse and open encounters in a calm normal mode, then intentionally
enter Manage Mode when they need administrative controls such as rename,
reorder, or upload.

## Non-Goals

- Do not redesign storage.
- Do not add new workflows beyond the mode separation.
- Do not add fog.
- Do not change Player Display behavior.
- Do not add campaign-management scope outside existing map-backed encounters.

## Acceptance Notes

- Normal Mode supports browsing and opening encounters.
- Manage Mode exposes existing upload, rename, and reorder controls.
- Delete is deferred because there is no reviewed delete API or storage flow.
- Normal Mode remains uncluttered.
- Mode switching is clear and does not surprise the GM.
- Administrative controls are not prominent during normal browsing.
- `Show to Players` remains explicit and available according to the approved UX
  design.
- Opening an encounter in Normal Mode does not change the Player Display.
- Manage Mode applies only to the encounter gallery, not campaign cards.

## Validation Plan

Validate Normal Mode with zero, one, and multiple encounters. Confirm
`Manage Encounters` is visible, normal browsing shows thumbnails, encounter
names, status badges, open-for-prep actions, and explicit `Show to Players`, and
normal browsing hides upload, rename, reorder, and delete controls.

Enter Manage Mode and confirm `Done Managing`, upload, rename, and reorder
controls are available and keyboard-accessible. Confirm delete is absent. Rename
an encounter, reorder encounters, upload a valid encounter, and attempt an invalid
upload. Confirm invalid upload preserves encounter names, order, and shown state.

With Player Display open, confirm opening an encounter in Normal Mode and using
Manage Mode upload, rename, and reorder do not change the Player Display or
reload the player asset. Confirm Player Display changes only after explicit
`Show to Players`. Validate desktop, Chromebook-sized, and narrow widths for no
horizontal overflow and touch-friendly mode switching.
