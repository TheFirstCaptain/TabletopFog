# F-005H: Manage Mode

## Objective

Separate browsing from administration.

## User Value

The GM can browse and open encounters in a calm normal mode, then intentionally
enter Manage Mode when they need administrative controls such as rename,
delete, reorder, or upload.

## Non-Goals

- Do not redesign storage.
- Do not add new workflows beyond the mode separation.
- Do not add fog.
- Do not change Player Display behavior.
- Do not add campaign-management scope outside existing map-backed encounters.

## Acceptance Notes

- Normal Mode supports browsing and opening encounters.
- Manage Mode exposes rename, delete if already supported or explicitly
  approved in this feature's reviewed scope, reorder, and upload controls.
- Normal Mode remains uncluttered.
- Mode switching is clear and does not surprise the GM.
- Administrative controls are not prominent during normal browsing.
- `Show to Players` remains explicit and available according to the approved UX
  design.
- Opening an encounter in Normal Mode does not change the Player Display.

## Clarification Notes

- The user requested Manage Mode as a focused feature to separate browsing from
  administration.
- Delete is listed in the desired Manage Mode controls, but current implemented
  support must be confirmed during clarification before implementation. If
  delete is not already supported, adding it may require separate feature scope.
- Align with `docs/design-language.md`: Progressive Disclosure, Calm Interface,
  Editing Is Different From Running, and Rule of Three.

## Affected Systems

- GM View: encounter browsing mode, Manage Mode controls, and mode switching.
- Player Display: no intended changes.
- Local server: no intended changes unless approved scope includes delete.
- Map/fog rendering: no intended changes.
- Storage: no intended changes unless approved scope includes delete.
- Documentation: update workflow and acceptance notes for browsing versus
  administration.

## Validation Plan

Validate Normal Mode with multiple encounters and confirm browsing/opening stays
uncluttered. Enter Manage Mode and confirm approved administrative controls are
available, keyboard-accessible, and do not affect the Player Display unless the
GM explicitly uses `Show to Players`. If delete enters scope, validate
data-integrity failure paths before implementation can be marked complete.

## Open Questions

- Is delete already supported enough to expose, or should it become a separate
  future feature?
- Should `Show to Players` remain visible in Normal Mode, Manage Mode, or both?
- Should Manage Mode apply only to encounter galleries or also to campaign
  cards later?
