# F-005I: Design Language Compliance Review

## Objective

Review the implemented UI against `docs/design-language.md`.

## User Value

Future UI work starts from a clear, documented assessment of where the current
application supports the product philosophy and where it drifts toward clutter,
surprise, or VTT-like complexity.

## Non-Goals

- Do not implement functionality changes.
- Do not implement visual changes.
- Do not change application code.
- Do not change tests.
- Do not add new product scope.

## Acceptance Notes

- Review current UI compliance with One Question Per Screen.
- Review current UI compliance with Calm Interface.
- Review current UI compliance with Progressive Disclosure.
- Review current UI compliance with Maps First.
- Review current UI compliance with Never Surprise the GM.
- Review current UI compliance with Editing Is Different From Running.
- Review current UI compliance with Cards Over Tables.
- Review current UI compliance with Rule of Three.
- Review current UI compliance with Respect The Table.
- Identify UI that conflicts with these principles.
- Recommend changes without implementing functionality changes.
- Document any recommendations as follow-up features, bug reports, or
  acceptance-note updates as appropriate.
