# F-005D: Navigation Simplification

## Objective

Reduce navigation clutter and remove implementation-focused information from the
GM UI.

## User Value

The GM can move between Campaign Library, Campaign, and Encounter Workspace
screens without parsing technical details or competing navigation affordances.

## Non-Goals

- Do not redesign page layouts.
- Do not redesign campaign cards.
- Do not redesign encounter cards.
- Do not add fog.
- Do not add new functionality.
- Do not change storage, APIs, or campaign data.

## Acceptance Notes

- Filesystem paths are removed from visible normal GM UI except explicit
  troubleshooting or diagnostics contexts.
- Campaign Library, Campaign, and Encounter Workspace use consistent
  breadcrumbs.
- Page headers are simplified and avoid implementation-focused labels.
- Every page has one obvious Back action.
- Page titles are standardized across the GM workflow.
- Navigation consistency is reviewed across Campaign Library, Campaign, and
  Encounter Workspace.
- Opening or editing an encounter still does not change the Player Display.
- `Show to Players` remains the only explicit action that updates the Player
  Display.
