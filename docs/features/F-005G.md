# F-005G: Encounter Workspace Layout

## Objective

Redesign the encounter workspace around the map.

## User Value

The GM gets a focused encounter surface where the map is prominent, the shown
status is obvious, and future tools have a clear home without making the screen
feel temporary.

## Non-Goals

- Do not add fog tools.
- Do not change canvas behavior.
- Do not change rendering behavior.
- Do not add new functionality.
- Do not change storage or APIs.

## Acceptance Notes

- The map is more prominent in the Encounter Workspace.
- Information hierarchy makes selected/editing encounter identity clear.
- `Show to Players` placement is obvious without becoming surprising or
  overbearing.
- Shown/editing status presentation is clear and uses `Shown to Players`
  terminology consistently.
- A dedicated future tool dock is reserved without adding real fog controls.
- Placeholder feel is removed from the workspace.
- Workspace layout remains usable on Chromebook-sized screens.
- Opening or editing the workspace still does not change the Player Display.

## Clarification Notes

- This feature is layout-only for the Encounter Workspace shell established in
  F-005C.
- Future tool dock means a stable reserved region for later tools, not hidden
  fog functionality.
- Align with `docs/design-language.md`: The Map Is The Hero, Maps First,
  Editing Is Different From Running, and Never Surprise the GM.

## Affected Systems

- GM View: Encounter Workspace layout, hierarchy, status, and future tool dock.
- Player Display: no intended changes except existing explicit `Show to Players`
  behavior.
- Local server: no intended changes.
- Map/fog rendering: no rendering behavior changes.
- Storage: no intended changes.
- Documentation: update workspace layout expectations if needed.

## Validation Plan

Validate the workspace with one shown encounter and a different selected/editing
encounter. Confirm map prominence, status clarity, future tool dock placement,
Back to Encounters behavior, and workspace `Show to Players` behavior. Confirm
opening or navigating the workspace does not update the Player Display.

## Open Questions

- Should the future tool dock be lateral, bottom-aligned, or responsive by
  viewport?
- Should `Show to Players` live near the title, near the map, or in a dedicated
  running-actions area?
