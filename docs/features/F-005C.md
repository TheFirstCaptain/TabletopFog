# F-005C: Encounter Workspace Shell

## Objective

Create the GM-facing workspace shell for editing or prepping a single encounter before fog tools are added in F-006.

## User Value

The GM gets a focused prep surface for one encounter while retaining explicit control over which encounter is shown to players during the session.

## Non-Goals

- Do not implement fog controls in F-005C.
- Do not implement initiative tracking.
- Do not implement notes.
- Do not add tokens.
- Do not add dynamic lighting.
- Do not add a full VTT toolbar.
- Do not rename storage or APIs without a reviewed migration plan.

## Acceptance Notes

- GM can open an encounter workspace from the encounter gallery.
- Workspace shows the selected/editing encounter name.
- Workspace shows the map using the existing canvas renderer.
- Workspace includes clear navigation back to the campaign encounter gallery.
- Workspace shows whether this encounter is currently shown to players.
- Workspace includes an explicit `Show to Players` action.
- Workspace reserves space for future fog controls without implementing them.
- The GM can inspect or prep one encounter while another remains shown to players.
- Player view remains unchanged unless `Show to Players` is clicked.
- No actual fog drawing tools are included unless separately approved in F-006.

## Terminology Notes

- The selected/editing encounter is the encounter currently open in the GM workspace.
- The shown-to-players encounter is the encounter currently displayed on `/player`.
- Opening a workspace must not mutate the shown-to-players encounter.
- Existing `activeMapId` storage/API terminology currently represents the shown-to-players map, not necessarily the selected/editing encounter.
- A future migration to clearer terminology such as `shownMapId` may be considered after review, but F-005C should not migrate storage by default.

## Clarification Notes

- Keep F-005C narrower than F-005B by building on the existing encounter gallery and GM-local selected encounter state.
- Turn the F-005B workspace preview into a focused in-page workspace shell rather than adding a route, modal, or persisted workspace state.
- Keep rename and reorder in the encounter gallery for F-005C; do not move map management into the workspace.
- Mirror the explicit `Show to Players` action in the workspace while preserving the card-level action from F-005B.
- Keep the workspace simple: selected encounter name, existing canvas map view, shown/not-shown status, gallery navigation, explicit show action, and empty reserved space for later fog tools.
- Do not add fog tools, notes, tokens, initiative, dynamic lighting, or a VTT-style toolbar.
