# F-005J: Final UI Polish

## Objective

Perform final visual polish before beginning F-006.

## User Value

The GM starts fog work from a UI that already feels coherent, calm, and ready
for game-night use, reducing the chance that fog tools are built on top of
avoidable presentation inconsistencies.

## Non-Goals

- Do not change functionality.
- Do not add fog tools.
- Do not change canvas behavior.
- Do not change storage.
- Do not add new workflows.
- Do not expand TabletopFog toward full VTT scope.

## Acceptance Notes

- Typography is consistent across Campaign Library, Campaign, Encounter
  Workspace, and Player Display.
- Buttons use consistent hierarchy, labels, spacing, hover states, and focus
  states.
- Spacing and alignment are polished across supported viewports.
- Empty states are clear and calm.
- Loading states are consistent and unobtrusive.
- Error presentation is readable and does not dominate normal workflows.
- Responsive cleanup covers desktop, Chromebook-sized, and narrow viewports.
- No animation or transition polish is introduced.
- Overall visual consistency is improved without changing behavior.
- No functionality changes are introduced.

## Feature Readiness Checklist

- Clarification complete.

- UX and workflow reviewed.
- Non-goals confirmed.

## Clarification Notes

- This is the final focused UI polish pass before F-006 Manual Fog of War.
- Use F-005I findings as input when available, but keep this feature to visual
  polish and consistency.
- Do not use this feature to sneak in navigation restructuring, Manage Mode, or
  new controls.
- Avoid animations and transitions entirely for this pass, including hover,
  focus, and mode changes.
- Include purely visual cleanup for the Encounter Workspace `Future Tools`
  placeholder so it competes less with the map without adding controls or
  changing workspace behavior.
- Include affordance polish for `Shown to Players` so the current shown
  encounter's clear action remains visibly actionable rather than looking like a
  passive badge.
- Include Player Display chrome density polish while preserving read-only
  behavior and keeping zoom/fit controls visually subordinate to the map.
- Include encounter-centered delete-confirmation copy polish, replacing
  implementation-ish language such as `map file` without changing deletion
  behavior.
- Exclude Campaign Library workflow changes, including a `Manage Campaigns`
  mode. Treat that as a separate future feature only if the current Campaign
  Library administration controls remain too busy in use.
