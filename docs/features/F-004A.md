# F-004A: Fantasy Visual Theme

## Objective

Give the existing campaign and map views a restrained tabletop-fantasy look
through a locally available Garamond-style typeface and an accessible
parchment-inspired color palette.

## User Value

The GM and players see an interface that feels appropriate for a fantasy
tabletop session while remaining readable, lightweight, and fully usable
without an internet connection.

## Non-Goals

- Do not redesign the campaign or map workflows.
- Do not add controls, animations, decorative imagery, or textured background
  assets.
- Do not change map scaling, fog behavior, or player synchronization.
- Do not require Adobe Fonts, Google Fonts, or another hosted runtime service.
- Do not use a font without clear redistribution and self-hosting rights.

## Acceptance Notes

- Remove `public/assets/fonts/Middleearth-ao6m.ttf` from the repository.
- Replace the current system-only visual treatment with a readable
  Garamond-style fantasy treatment.
- Prefer a locally bundled, web-optimized open font such as EB Garamond, with
  its license recorded alongside the asset.
- Do not bundle Minion Pro unless the user supplies explicit self-hosting and
  redistribution rights.
- Use explicit fallback fonts so the interface remains usable if the custom
  font fails to load.
- Apply a parchment-inspired palette with sufficient contrast for text,
  controls, focus states, status messages, and diagnostics.
- Preserve a dark player map stage unless clarification explicitly chooses a
  parchment stage and validation shows that maps retain adequate contrast.
- Preserve all current GM and player labels, controls, layout behavior,
  read-only boundaries, and Socket.IO synchronization.
- Validate desktop and narrow/mobile layouts in Chromium and retain physical
  Safari/iPad checks separately.

## Clarification Notes

- The user requested a fantasy look using Minion Pro or Garamond and a
  parchment-paper background color.
- This is tracked as F-004A because it is a visual follow-up to F-004's campaign
  and map views; F-002 owns the completed local-connectivity spike.
- The unused Middleearth TTF was not referenced by the stylesheet and was
  removed during implementation.
- EB Garamond is the recommended starting point because it is available under
  the SIL Open Font License and can be bundled for offline local hosting.
- Adobe Fonts does not permit self-hosting its supplied web font files, so
  Minion Pro is not the default implementation candidate.
- Use EB Garamond for headings and display labels while retaining the existing
  system-font stack for controls and dense text.
- Apply parchment styling to the GM workspace and player status bar while
  preserving the dark player map stage.
- Use subtle CSS-only tonal variation without a texture image.
- Bundle the Google Fonts v32 Latin variable WOFF2 for normal weights 400
  through 800 as `EBGaramond-Variable-Latin.woff2`; its SHA-256 is
  `79d17b52365a2d5bd8995c8c54939d384e9888ed2038c7201fd8d4118d6f0a35`.
- Bundle the upstream `OFL.txt` and a `SOURCE.md` recording the asset URL,
  weight range, Latin unicode range, retrieval date, and checksum. Do not
  convert, subset, or modify the upstream WOFF2.
- Use the reviewed palette: page `#e8d8b5`, panel `#f7ebcf`, strong surface
  `#e3d0a8`, text `#2b2118`, muted `#685843`, border `#9b7b4f`, accent
  `#6b3e24`, accent strong `#4a2918`, danger `#8c1d2c`, danger surface
  `#f5d8d2`, focus `#1f5e73`, player stage `#14110f`, and player stage text
  `#c9b17f`.
- Require at least 4.5:1 contrast for normal text, statuses, diagnostics, and
  control labels, and at least 3:1 for large text, focus indicators, and
  meaningful control boundaries.
