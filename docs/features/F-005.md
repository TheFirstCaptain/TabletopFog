# F-005: Static Active Map Display Polish

## Objective

Harden and polish the selected active campaign map display on both GM and player views after F-004's minimal display path exists.

## User Value

The GM and players get a more reliable table display before fog-of-war controls are added.

## Non-Goals

- Do not add fog controls.
- Do not add new campaign library behavior beyond F-004.
- Do not add tokens.
- Do not add notes.
- Do not add initiative tracking.

## Acceptance Notes

- GM and player views use consistent active-map rendering behavior.
- Map scales reasonably to the available screen across desktop, iPad, and TV-like landscape viewports.
- Active-map display handles refresh, reconnect, and image-load error states cleanly.
- Player receives only the active-map display state needed for read-only rendering.
- Player view remains read-only.
- No fog controls are exposed yet.

## Affected Systems

- GM view: active-map display polish.
- Player view: read-only active-map display polish.
- Local server: active-map sync hardening if needed.
- Map/fog rendering: static map rendering and scaling.
- Storage: reads campaign map metadata and assets created by F-004.
- Documentation: update acceptance tests for active map display.

## Validation Plan

Open a campaign with multiple maps, select each map from the GM view, refresh/reconnect GM and player views, and verify both views show the same active map with reasonable scaling on desktop and iPad.

## Open Questions

- Should the GM view use the exact same renderer as the player view with controls around it?
- What fit mode should be the default for wide maps on iPad mirrored to a TV?
- What display fit modes are needed beyond default contain behavior?

## Status

Proposed.
