# F-005: Static Active Map Display

## Objective

Display the selected active campaign map on both GM and player views.

## User Value

The GM and players can see the same selected campaign map before fog-of-war controls are added.

## Non-Goals

- Do not add fog controls.
- Do not add save/load beyond what F-004 needs to open the active campaign.
- Do not add tokens.
- Do not add notes.
- Do not add initiative tracking.

## Acceptance Notes

- GM can select a map from the campaign.
- Player view updates to show the selected active map.
- Map scales reasonably to the available screen.
- GM and player views show the same active map.
- Player view remains read-only.
- No fog controls are exposed yet.

## Affected Systems

- GM view: active-map selection and map display.
- Player view: read-only active-map display.
- Local server: syncs active-map changes.
- Map/fog rendering: static map rendering and scaling.
- Storage: reads campaign map metadata and assets created by F-004.
- Documentation: update acceptance tests for active map display.

## Validation Plan

Open a campaign with multiple maps, select each map from the GM view, and verify both GM and player views update to the same active map on desktop and iPad.

## Open Questions

- Should the GM view use the exact same renderer as the player view with controls around it?
- What fit mode should be the default for wide maps on iPad mirrored to a TV?
- Should active-map changes be persisted immediately or only when F-007 save/load is implemented?

## Status

Proposed.
