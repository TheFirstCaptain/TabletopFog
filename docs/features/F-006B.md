# F-006B: Fog State and Rendering Foundation

## Objective

Establish the V1 manual-fog foundation: encounter-scoped in-memory fog state,
ordered hide/reveal operation semantics, and shared fog rendering in the GM
View and Player Display.

## User Value

The GM and players see the same fog model rendered appropriately for their role
before editing tools are layered on top. This reduces risk before drawing,
revealing, clearing, and persistence work.

## Non-Goals

- Do not add GM drawing tools yet.
- Do not add circles, brushes, undo, reset, or shape editing.
- Do not persist fog to campaign storage.
- Do not migrate storage or API terminology.
- Do not add dynamic lighting, tokens, vision cones, notes, or initiative.
- Do not send fog for encounters that are not shown to players.

## Acceptance Notes

- Fog state is stored in memory per encounter for the current server process.
- The V1 fog state is an ordered operation list.
- Fog rectangles use normalized map-relative coordinates, not screen pixels, so
  rendering stays stable across GM zoom, player zoom, viewport size, and later
  persistence.
- Supported operation types at this layer are `hide-rectangle` and
  `reveal-rectangle`.
- `reveal-rectangle` exists in F-006B only as state, rendering, and validation
  behavior; user-facing Reveal controls remain F-006D.
- Rendering replays fog operations in order.
- Reveal operations visually remove fog inside their rectangle.
- Later hide operations can cover an area revealed by an earlier reveal
  operation.
- GM View renders fog semi-transparently over the selected encounter map.
- Player Display renders fog opaquely or near-opaquely over the shown-to-players
  encounter map.
- Player Display receives fog state only for the encounter currently shown to
  players.
- Editing or preparing fog state for an unshown encounter must not change the
  Player Display.
- Maps still start visible by default.
- Validation may seed fog through internal test fixtures or direct harness setup,
  but F-006B must not add visible temporary controls, debug UI, or a new
  developer-only route just to create fog.
