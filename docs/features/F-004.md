# F-004: Static Map Display

## Objective

Allow the GM to choose or load a static map image and show that same map on the player display.

## User Value

The app becomes useful as a basic shared battlemap display before fog-of-war controls are added.

## Non-Goals

- Do not add tokens.
- Do not add fog reveal.
- Do not add persistent sessions unless required for basic browser behavior.
- Do not add map libraries or complex scene management.

## Acceptance Notes

- GM can choose or load a map image.
- Player view displays the same map.
- Map scales reasonably to the available screen.
- Player view remains read-only.

## Affected Systems

- GM view: map selection or loading control.
- Player view: map display only.
- Local server: distributes selected map state as needed.
- Map/fog rendering: static map rendering.
- Storage: none or temporary browser/server memory.
- Documentation: update acceptance tests for map display.

## Validation Plan

Load a representative map image, verify GM and player views show the same image, and test resizing on desktop and iPad.

## Open Questions

- Should the first map source be a local file input, a sample asset, or a served upload?
- How should large image scaling be constrained on iPad?

## Status

Proposed.
