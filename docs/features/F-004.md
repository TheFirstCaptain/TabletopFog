# F-004: Basic Fog of War

## Objective

Add manual fog-of-war reveal over a static map.

## User Value

The GM can hide unrevealed map areas while showing only discovered spaces to players.

## Non-Goals

- Do not add dynamic lighting.
- Do not add tokens or vision cones.
- Do not add automation based on rules, NPCs, or character stats.
- Do not add secret room layers.

## Acceptance Notes

- Full black fog layer starts over the map.
- GM can reveal simple shapes such as rectangles or circles.
- Player sees only revealed areas.
- Hidden areas remain hidden on the player display.
- Player view remains read-only.

## Affected Systems

- GM view: simple reveal tool.
- Player view: revealed-only map rendering.
- Local server: syncs reveal state.
- Map/fog rendering: canvas map and fog layers.
- Storage: in-memory reveal state until F-005.
- Documentation: update fog correctness tests.

## Validation Plan

Start with full fog, reveal simple shapes from the GM view, and compare GM/player views on desktop and iPad.

## Open Questions

- Store reveals as vector shapes first or rasterize a fog mask?
- Should GM view show hidden areas differently from player view?

## Status

Proposed.
