# F-005K: Clear Shown Encounter

## Objective

Let the GM clear the encounter currently shown to players.

## User Value

The GM can stop showing an encounter without choosing a replacement, returning
the Player Display to its waiting state between scenes or during table resets.

## Non-Goals

- Do not add a confirmation flow.
- Do not add delete or archive behavior.
- Do not change encounter storage beyond clearing the shown-to-players identity.
- Do not change map rendering or fog behavior.
- Do not add Player Display controls.

## Acceptance Notes

- A currently shown encounter has an enabled `Shown to Players` action.
- Clicking `Shown to Players` clears the Player Display.
- The cleared encounter action changes back to `Show to Players`.
- The workspace running action also clears when the selected encounter is
  currently shown.
- Status badges no longer mark any encounter as `Shown to Players`.
- The Player Display returns to its waiting state.
- Opening or selecting encounters remains GM-local unless `Show to Players` or
  `Shown to Players` is clicked.

## Clarification Notes

- The user requested clearing the current shown-to-players encounter by clicking
  the existing `Shown to Players` button.
- The button should return to `Show to Players` after the encounter is unshown.
- Clearing means no encounter is shown to players; it does not delete, archive,
  reorder, rename, or otherwise edit the encounter.
- Clearing is represented by persisted `activeMapId: null` until a future naming
  migration replaces the active-map implementation term.
- The existing GM-only active-map endpoint should accept `{ "mapId": null }` as
  the explicit clear request and continue rejecting missing, unknown, or invalid
  map identifiers.
- The enabled `Shown to Players` button is an intentional toggle exception to
  the normal terminology rule; accessible names should make the clear action
  explicit.
- No confirmation is needed because clearing is reversible and should stay fast
  at the table.
