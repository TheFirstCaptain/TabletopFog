# F-006: Manual Fog of War

## Objective

Add manual hide/reveal fog controls over a visible map.

## User Value

The GM can hide unrevealed or secret areas of an otherwise visible map, then remove fog during play as players discover those areas.

## Non-Goals

- Do not start maps under a full black fog layer.
- Do not add dynamic lighting.
- Do not add tokens or vision cones.
- Do not add automation based on rules, NPCs, or character stats.
- Do not add secret room layers.
- Do not add multiple fog states per map.

## Acceptance Notes

- Map starts visible.
- GM can add fog or hide areas using simple shapes such as rectangles or circles.
- GM can remove or reveal fog from hidden areas.
- Player sees the same map with hidden areas obscured.
- Hidden areas remain hidden on the player display.
- Player view remains read-only.
- Fog state belongs to a single map.
- No multiple fog states per map.
- No dynamic lighting.
- No tokens or vision cones.

## Affected Systems

- GM view: simple hide and reveal tools.
- Player view: read-only rendering with hidden areas obscured.
- Local server: syncs fog state for the active map.
- Map/fog rendering: visible map plus fog shapes or mask.
- Storage: in-memory fog state until F-007 persistence.
- Documentation: update fog correctness tests and storage-shape notes.

## Validation Plan

Start from a visible map, add simple fog shapes from the GM view, remove or reveal part of the fog, and compare GM/player views on desktop and iPad.

## Open Questions

- Store fog as vector hide/reveal operations, normalized shapes, or a rasterized fog mask?
- Should GM view show fog as semi-transparent while player view shows it opaque?
- How should reveal operations interact with overlapping fog shapes?

## Status

Proposed.
