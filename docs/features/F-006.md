# F-006: Table Quality-of-Life Improvements

## Objective

Improve the table experience after local connectivity, map display, fog reveal, and save/load are proven.

## User Value

The GM can run sessions with less friction once the core workflow works.

## Non-Goals

- Do not convert TabletopFog into a full VTT.
- Do not add initiative tracking unless it is split into a future approved feature.
- Do not add dynamic lighting or secret room layers in this feature.
- Do not add cloud dependencies.

## Acceptance Notes

- Better reveal brush.
- Undo reveal.
- Clear fog or reset fog.
- Fullscreen player view.
- QR code or copied player URL.

## Affected Systems

- GM view: improved controls.
- Player view: fullscreen and display polish.
- Local server: may expose player URL details.
- Map/fog rendering: brush and undo behavior.
- Storage: may need undo-aware state representation.
- Documentation: split into child features before implementation if the scope grows.

## Validation Plan

Validate each improvement independently with manual GM/player checks and update automated tests when the implementation has stable behavior.

## Open Questions

- Which quality-of-life improvement should be first after save/load?
- Should QR code generation be client-side or server-side?

## Status

Proposed.
