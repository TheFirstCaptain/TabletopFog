# E-013: Split Fog-Only Workspace Rendering

## Summary

Avoid rebuilding the full encounter gallery when a fog-only change affects the
selected Encounter Workspace.

## Status

Proposed.

## Behavior to Preserve

- Campaign page encounter cards, thumbnails, status badges, and direct controls
  remain visually and functionally unchanged.
- Opening an encounter remains GM-local.
- Fog edits, Clear Fog, and Undo keep their current Player Display update
  boundaries.
- Recovery diagnostics and navigation behavior remain unchanged.

## Motivation

E-010 found that `renderCampaign()` always calls `renderMaps()`. Fog-only
workspace changes currently recreate all encounter cards and thumbnail elements
even though the gallery does not need to change for most fog edits. This is
low risk at current campaign sizes but becomes unnecessary DOM churn as
encounter count and thumbnail count grow.

## Scope

- In scope:
  - Identify state changes that can update only workspace controls/canvas.
  - Split or add a narrow render path for selected Encounter Workspace changes.
  - Keep full gallery rendering for campaign membership, ordering, name,
    selected, and shown-to-players changes.
- Out of scope:
  - Visual redesign.
  - Virtualized lists.
  - Changing card controls or workflow behavior.

## Affected Systems

- GM view: `public/gm-view.js` and `public/gm-controller.js` render paths.
- Player view: no direct change.
- Local server: no expected change.
- Storage: no change.
- Tests and tooling: focused Chromium regression coverage for fog changes,
  selected/shown status, and card controls.
- Documentation: this engineering record and tracker.

## Implementation Notes

Keep the existing full-render path as the conservative fallback. Add the narrow
path only where state transitions are easy to classify, such as successful fog
append, Clear Fog, or Undo for the selected encounter.

## Validation Plan

- Chromium coverage for fog append, Clear Fog, and Undo after the split.
- Existing encounter gallery and workspace tests.
- `npm run quality`.
