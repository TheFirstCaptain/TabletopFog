# E-007: Stabilize Browser Reconnect Timing

## Summary

Allow the Chromium reconnect characterization enough time for Socket.IO's
normal reconnection backoff when it runs after the expanded browser suite.

## Status

Done.

## Behavior to Preserve

After a temporary offline period, the player returns to `Live` and restores the
same active map without a page reload.

## Motivation

The reconnect test passes immediately in isolation but timed out twice in the
complete six-test order because its default 5-second visibility assertion is
too close to the reconnection/backoff boundary. This makes the authoritative
quality command nondeterministic without identifying a product regression.

## Scope

- In scope: increase only the reconnect test and post-online assertion timing
  budget, preserving all existing outcome assertions.
- Out of scope: production Socket.IO settings, reconnect behavior, UI messages,
  retries for unrelated tests, or global test timeouts.

## Affected Systems

- Player view: no production change.
- Tests and tooling: one Chromium reconnect characterization timeout.
- Documentation: engineering tracker and F-004A validation evidence.

## Implementation Notes

Keep the offline assertion, restore the context online, allow up to 15 seconds
for the existing `Live` outcome, and give this test a 30-second total budget.
Do not add retries or weaken active-map restoration and decoded-image checks.

## Validation Plan

Run the reconnect test in isolation, the complete six-test Chromium suite, and
`npm run quality`. Confirm the player still reconnects without reload and the
same active image is visible and decoded.

## Resolution

Only the reconnect test now has a 30-second total budget, and only its
post-online `Live` assertion allows up to 15 seconds. Offline detection,
no-reload behavior, active-image restoration, and decoded-image assertions are
unchanged. No production Socket.IO or player-view code changed.

## Validation Evidence

- The reconnect workflow passed in isolation after the scoped timing change.
- The complete six-test Chromium suite passed with all reconnect and image
  restoration outcomes retained.
- `npm run quality` passed lint, formatting, the 20-module ratchet, harness
  compliance, all 75 Node tests with coverage floors, all 6 Chromium tests, and
  a zero-vulnerability high-severity dependency audit.
