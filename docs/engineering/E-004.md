# E-004: Separate Server Transport Responsibilities

## Summary

Separate API routing, role-based state projection, Socket.IO coordination, and
HTTPS startup into focused server modules.

## Status

Done.

## Behavior to Preserve

Existing routes, response payloads, role behavior, Socket.IO events, HTTPS
startup, and dependency injection remain unchanged.

Detailed preservation contract:

- `createApp(options)` continues to return an Express app with the same route
  paths, middleware behavior, response status codes, JSON shapes, static asset
  serving, and error responses.
- `createTabletopFogServer(options)` continues to return `{ app, io, server,
  syncState, stateStore }`.
- `start()` continues to read `PORT` and `HOST`, load HTTPS credentials, listen
  on the configured host and port, and print the same GM and Player Display URL
  guidance.
- `getRoleFromReferer(referer)` continues to classify only `/gm` as `gm`; all
  missing, malformed, or non-GM referers remain `player`.
- `projectStateForRole(state, "gm")` continues to return full GM state, while
  player projection continues to include only `activeMap`, `updatedAt`, and
  `version`.
- Player projection continues to expose only the encounter currently represented
  by `activeMapId`, including its fog operations, and never exposes the full
  campaign or unshown encounter fog.
- GM-only route protection continues to reject player-originated mutations with
  the same `403` response shape.
- Campaign, map, active-map, fog append, fog clear, and asset routes keep their
  current side effects and ordering.
- Existing dependency injection seams for `campaignStorage`, `stateStore`,
  `credentials`, `dataRoot`, and `env` remain available to tests and local
  startup helpers.

## Motivation

`server/index.js` currently combines application construction, campaign routes,
error handling, socket projection, role interpretation, and process startup.
Further map and fog endpoints will make those boundaries harder to introduce
later.

This work is queued before F-007 so save/load behavior can use clearer route,
projection, Socket.IO, and startup boundaries.

## Planned Module Boundaries

Prefer small modules with narrow, named responsibilities instead of one large
server abstraction. A likely target shape:

- `server/http-routes.js`: register Express middleware, static files, GM page,
  Player Display page, API routes, route-level validation, and error handling.
- `server/role-projection.js`: own `getRoleFromReferer`, `requireGm`, and
  `projectStateForRole`.
- `server/socket-sync.js`: create Socket.IO coordination, assign socket roles,
  emit role-projected `state:sync`, and expose `syncState`.
- `server/index.js`: composes storage, state store, route registration,
  credentials, HTTPS server, Socket.IO, startup, and preserved public exports.

Do not introduce a framework, dependency, async router layer, authentication
model, or renamed API/storage terminology as part of E-004.

## Suggested Incremental Sequence

1. Extract Role Projection First

   Move role parsing, GM guard, and state projection into a focused module while
   preserving current exports from `server/index.js`. Add or retain direct unit
   tests for referer classification and player projection.

2. Extract Socket Synchronization

   Move `emitState`, Socket.IO `connection` handling, and the `syncState`
   closure into a small coordinator that accepts `io`, `stateStore`, and the
   projection helpers. Preserve `state:sync` event names and initial sync
   behavior.

3. Extract Route Registration

   Move route registration into a function that accepts `app`,
   `campaignStorage`, `stateStore`, `onStateChange`, and `publicDir`. Keep
   request validation and response shapes unchanged. Avoid changing storage
   calls or route order during this extraction.

4. Simplify Server Composition

   Leave `createApp`, `createTabletopFogServer`, and `start` as public
   composition functions. After extraction, `server/index.js` should read as a
   thin assembly layer plus exports.

5. Update Module Baseline Intentionally

   Add new modules and reduce `server/index.js` in
   `quality/module-baseline.json`. The E-004 temporary exception on
   `server/index.js` was removed after HTTP, role, socket, and startup
   responsibilities were separated enough to close the exception.
