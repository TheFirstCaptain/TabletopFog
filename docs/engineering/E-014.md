# E-014: Parallelize Browser Workflow Tests

## Summary

Review and, if stable locally and in CI, enable fully parallel Chromium workflow
tests to reduce validation runtime without weakening isolation or diagnostics.

## Status

Validating.

## Behavior to Preserve

- Browser tests still use isolated HTTPS servers, temporary campaign data,
  temporary certificates, fresh browser contexts, and deterministic cleanup.
- Failure artifacts remain useful.
- The authoritative `npm run quality` command remains the local and CI policy.
- No test coverage is removed or weakened.

## Motivation

E-010 measured the browser suite at about 21.1 seconds with the configured
single worker. A local run with `npm run test:browser -- --fully-parallel
--workers=4` passed and completed in about 7.7 seconds. The fixture design
already isolates server ports, data roots, certificates, and contexts, so this
is a plausible tooling speedup.

## Scope

- In scope:
  - Confirm parallel runs are stable across repeated local executions.
  - Confirm CI resource usage and diagnostics remain acceptable.
  - Update Playwright configuration if evidence supports the change.
- Out of scope:
  - Removing browser scenarios.
  - Changing app behavior.
  - Changing the Node test runner policy.

## Affected Systems

- GM view: no product change.
- Player view: no product change.
- Local server: exercised concurrently by isolated test servers.
- Storage: temporary data roots remain isolated.
- Tests and tooling: `playwright.config.js`, browser fixtures, and CI quality
  runtime.
- Documentation: development guidance if validation behavior changes.

## Implementation Notes

Start by repeating fully parallel browser runs. If stable, update
`playwright.config.js` deliberately and run full `npm run quality`. Watch for
port cleanup, certificate cleanup, trace output, and failure readability.

Implemented on 2026-07-10:

- Confirmed the browser fixture design isolates each test with its own HTTPS
  server on an ephemeral localhost port, temporary campaign data root,
  generated certificate, browser context, and cleanup.
- Repeated `npx playwright test --fully-parallel --workers=4` three times
  before changing configuration; all 31 Chromium workflow tests passed in about
  7.7s, 6.8s, and 7.8s.
- Updated `playwright.config.js` so the default `npm run test:browser` and
  `npm run quality` browser stage run fully parallel with 4 workers.
- Preserved all browser scenarios, artifacts, retries, timeout, project, trace,
  screenshot, and HTTPS error behavior. No application code or product workflow
  changed.
- README and development docs were reviewed; no update was needed because the
  documented commands stay the same and continue to use `npm run test:browser`
  and `npm run quality`.
