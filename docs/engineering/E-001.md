# E-001: Clean Up Test Temporary Directories

## Summary

Ensure automated tests remove temporary campaign data and certificate files
after each run.

## Status

Proposed.

## Behavior to Preserve

All existing automated tests continue to run independently and validate the same
application behavior.

## Motivation

Tests create directories under the operating system temporary root but do not
remove them. Repeated runs leave avoidable filesystem artifacts.

## Scope

- In scope: shared temporary-resource registration and reliable teardown.
- Out of scope: changing application storage behavior or certificate generation.

## Affected Systems

- Tests and tooling: `test/campaign-storage.test.js`, `test/cert.test.js`, and
  `test/server.test.js`.

## Validation Plan

Run `npm test`, confirm all tests pass, and confirm created test directories are
removed after success and failure paths where practical.
