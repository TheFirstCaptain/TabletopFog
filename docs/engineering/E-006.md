# E-006: Harden Test Fixtures and Failure-Path Coverage

## Summary

Improve existing test inputs and coverage conventions so tests challenge
application assumptions using representative data, negative cases, and
partial-failure assertions.

## Status

Done.

## Behavior to Preserve

All currently promised application behavior remains unchanged, and the existing
automated suite continues to validate the same successful workflows.

## Motivation

The engineering audit found tests that used arbitrary text bytes labeled as PNG
data, allowing invalid production behavior to pass. The B-003 fix introduced a
real PNG fixture and initial negative cases, but the broader suite has no
documented fixture-quality or boundary-matrix convention.

## Scope

- In scope: inventory external-input fixtures, replace unrealistic inputs,
  define relevant valid/invalid/mismatched/oversized/partial-failure matrices,
  verify rejected operations leave no partial state, and share safe fixture
  helpers.
- Out of scope: changing product behavior, adding F-005/F-006 behavior, choosing
  browser automation, or duplicating E-001 resource cleanup.

## Affected Systems

- GM view: representative upload and workflow inputs where applicable.
- Local server: HTTP, storage, certificate, and Socket.IO failure paths.
- Storage: file and metadata rollback assertions.
- Tests and tooling: fixtures, helpers, and test-review conventions.
- Documentation: test guidance and F-009B review checklists after implementation.

## Implementation Notes

- Derive boundary matrices from the actual trust and persistence boundary rather
  than applying every case mechanically.
- Prefer small committed fixtures or deterministic fixture builders.
- Assert both the returned error and the absence of partial files, metadata, or
  state changes.
- Cross-link newly discovered broken behavior as bugs rather than silently
  changing expectations inside maintenance work.

## Boundary Matrix

| Boundary | Representative valid cases | Rejected cases | No-partial-state assertion |
| --- | --- | --- | --- |
| Image signature and metadata | PNG, JPEG, GIF, and WebP fixtures | Empty, arbitrary, oversized, extension mismatch, and content-type mismatch | Validation completes before storage writes. |
| Campaign storage mutations | Valid map order and active map | Missing, duplicate, or unknown map IDs | Campaign metadata remains byte-for-byte unchanged. |
| HTTP upload and JSON input | Valid PNG upload and campaign request | Invalid image, mismatched metadata, malformed JSON, and player mutation | Files, campaign metadata, and in-memory state remain unchanged. |
| HTTPS certificates | Generated certificate with requested SAN | Missing, malformed, or missing requested SAN | Regeneration decision only; no production certificate write occurs. |
| Socket and role projection | GM and player referers | Missing or malformed referer defaults to player | Player projection and read-only behavior remain unchanged. |

## Implementation Plan

- Expand deterministic fixture builders in `test-support/` instead of embedding
  ad hoc byte strings in tests.
- Add table-driven unit checks for every supported image type and relevant
  rejection class, including the documented 100 MB boundary.
- Add storage and HTTPS integration assertions that rejected operations leave
  metadata, files, and in-memory state unchanged.
- Document the fixture-quality and failure-path convention for future tests.
- Do not change production modules unless a test exposes broken promised
  behavior; reclassify such work as a bug first.

## Validation Plan

Run focused image, campaign storage, certificate, and HTTPS integration tests,
then `npm run quality`. Confirm each relevant matrix row has representative
valid and invalid coverage, rejected operations retain prior state, and E-001
removes all created test resources.

## Implementation Evidence

- Expanded `test-support/fixtures.js` with named PNG, JPEG, GIF, and WebP
  signature fixtures matching production validation depth plus a deterministic
  oversized-input builder.
- Added a shared generated HTTPS certificate helper with registered teardown.
- Added image validation coverage for every supported signature, content-type
  parameters, empty/arbitrary/oversized input, and extension/content-type
  mismatches.
- Added storage mutation matrices proving missing, unknown, duplicate, and
  empty inputs leave campaign metadata byte-for-byte unchanged.
- Added HTTPS integration matrices proving invalid and mismatched uploads leave
  files, metadata, and in-memory state unchanged, and malformed JSON creates no
  filesystem or state changes.
- Added certificate coverage for valid SANs, missing SANs, missing files, and
  malformed certificates; existing role tests cover missing and malformed
  referers defaulting to the read-only player role.
- Documented fixture realism, partial-state assertions, and cleanup conventions
  in AGENTS, development guidance, and the feature review template.
- Focused suites passed and `npm run quality` passed 68 tests with 85.87% lines,
  83.59% branches, 87.71% functions, and zero audit vulnerabilities.
- No broken promised behavior was discovered, so no bug record was required.
  README and acceptance tests required no update because commands and
  user-visible behavior are unchanged.
